# Многошаговые операции над Google Sheets (слияние позиций)

Слияние одной позиции каталога в другую = перепривязать её код во ВСЕХ листах
данных (PlantStock, LipStock, LipBatches, Inbound, Need) и в синонимах (лист
Aliases, два формата строк), добавить прежнее имя синонимом, удалить из Syryo.

## Правило: не глушить ошибки чтения листов
**Why:** наблюдался молчаливый частичный сбой — первый прогон вернул `ok:true`,
но синоним не перепривязался и source не удалён (вероятно транзиентный ПУСТОЙ
ответ Google на поздние чтения в длинной цепочке из ~15 последовательных
вызовов). `try/catch { return 0 }` вокруг чтения превращает такой сбой в «0
изменений» без ошибки → осиротевшие строки старого кода.
**How to apply:** эти листы всегда существуют (ensureSheets), поэтому сбой чтения
— реальный transient; пусть пробрасывается и прерывает операцию ГРОМКО. Плюс
явные sanity-guard: пустой Aliases → throw; deleteRawMaterial вернул false → throw.

## Правило: неидемпотентные записи (суммирование avg) делать ПОСЛЕДНИМ шагом
**Why:** при частичном сбое source остаётся в каталоге; повторный запуск снова
суммирует avg(target+source) → двойной учёт расхода.
**How to apply:** заготовь новую строку target в начале (по первому чтению), но
ЗАПИШИ её после удаления source. Тогда: если упало до delete — source ещё цел,
target не тронут, повтор пересчитает корректно; если упало после delete —
повтор бросит «source не найдена» (guard в начале) и не задвоит.

## Идемпотентность повтора
После успешного слияния source удалён → повторный POST /api/raw-materials/merge
бросает «Позиция X не найдена в каталоге». Это ожидаемо и защищает от двойного
применения.
