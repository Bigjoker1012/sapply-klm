---
name: Legacy payload contract on rewrites
description: При переписывании бэкенда сначала grep-нуть фронт на точные имена полей в JSON-ответе, иначе UI молча покажет пустоту.
---

При замене сторадж-слоя (Sheets → SQLite, REST → новый REST и т.п.) типизация на бэке не ловит расхождение с фронтом — фронт принимает `any` через axios и просто рендерит `undefined` как пусто. Архитект-ревью уже один раз поймало это: я заменил `raw_name`/`destination`/`original_text`/`source_type`/`file_name` на свои «логичные» `name`/`supplier`/`text`/`source`/`job` — фронт собрался, эндпоинт ответил 200, но в «сырьё в пути» и «требует проверки» отображались бы пустые строки.

**Правило перед коммитом любого route-rewrite:**
1. `rg -n "data\.<endpoint>|axios.get.*<endpoint>" client/` — где используется ответ.
2. Найти TS-интерфейс (если есть) или прямо в JSX/use-* читать конкретные поля.
3. Сохранить **точные** имена полей и тип id (часто string на фронте, integer в БД — кастовать через `String(r.id)`).
4. Прокомментировать DTO как «форма ровно та, что ждёт <ComponentName>» — следующий рефакторер не сломает.

**Why:** axios + ts-on-frontend-only = тихая регрессия. TypeScript сервера не знает контракт фронта. Тесты бэка проходят, UI выглядит сломанным.

**How to apply:** при переписывании любого `/api/*`-роута, обслуживающего существующий UI, обязательная проверка — grep по client/ + сверка с TS-интерфейсом фронта перед запуском.
