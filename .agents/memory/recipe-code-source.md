# Рецепт опознают по КОДУ, а не по описанию/заводу

Пользователь узнаёт рецепт по коду вида `Д-П60-3/Б20/ПЛЦ-NNN`. В списке «Рецепты»
код должен быть основной строкой; описание — вторичным. Наименование завода-
заказчика («ОАО …комбинат хлебопродуктов») НИКОГДА не должно становиться именем
рецепта.

**Где брать код (по надёжности):**
1. Текстовый слой PDF — там код обычно есть.
2. Имя загруженного файла — на проекте файлы названы по коду, поэтому это надёжный
   фолбэк, когда в шапке Excel технолога кода нет (а есть только завод-заказчик).

**Why:** Excel-разбор брал шапку-завод как имя и не извлекал код (старый regex
обрывался на «/»), из-за чего все рецепты в списке выглядели одинаково — как завод.

**How to apply:** при правках разбора рецептов держать приоритет «код → описание»,
фолбэк кода из имени файла, и отсекать орг-наименования из имени. Для уже
загруженных рецептов код восстанавливается из колонки file_name тем же фолбэком.
Массовый repair листа делать выборочно по строкам, а не перезаписью всего
диапазона (Sheets shared dev↔prod, last-write-wins под конкурентными правками).
